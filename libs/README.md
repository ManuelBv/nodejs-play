# libs

Practice files for CodeSignal-style interviews and DSA pattern drilling.

## Pattern drills (from [DSA-Interview-Playbook](https://github.com/sautrikroy17/DSA-Interview-Playbook), Step 2)

- [sliding-window.ts](sliding-window.ts) / [sliding-window.test.ts](sliding-window.test.ts) — 7 sliding window problems (fixed and variable window size).
- [two-pointers.ts](two-pointers.ts) / [two-pointers.test.ts](two-pointers.test.ts) — 8 two pointers problems (converging pointers, pair/triplet sums, in-place partitioning).
- [fast-slow-pointers.ts](fast-slow-pointers.ts) / [fast-slow-pointers.test.ts](fast-slow-pointers.test.ts) — 6 fast & slow pointers problems on linked lists (cycle detection, midpoint, palindrome checks).
- [merge-intervals.ts](merge-intervals.ts) / [merge-intervals.test.ts](merge-intervals.test.ts) — 6 merge intervals problems (overlap merging, insertion, meeting room scheduling).
- [binary-search.ts](binary-search.ts) / [binary-search.test.ts](binary-search.test.ts) — 7 modified binary search problems (boundaries, rotated arrays, search-on-the-answer).

## Other practice files

- [drills.ts](drills.ts) / [drills.test.ts](drills.test.ts) — array/string pattern-recognition drills plus a DFS/binary-tree reference walkthrough.
- [solutions.ts](solutions.ts) / [solutions.test.ts](solutions.test.ts) — 4 standalone warm-up problems in the style of CodeSignal's General Coding Assessment (GCA).
- [icf.ts](icf.ts) / [icf.test.ts](icf.test.ts) — a 4-level progressive exercise (Library Tracker) mirroring CodeSignal's Industry Coding Framework format.
- [codeChecker.ts](codeChecker.ts) / [codeChecker.test.ts](codeChecker.test.ts) — validates whether a given string is a valid UK postcode.
